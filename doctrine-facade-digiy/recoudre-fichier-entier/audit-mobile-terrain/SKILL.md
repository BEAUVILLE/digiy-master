Utilise cette skill pour toute page DIGIY qui doit être claire et utile sur téléphone.

## Objectif
S’assurer qu’un écran sert un geste métier concret immédiatement, sans surcharge.

## Contrôle prioritaire
- Le geste maître est visible dès l’ouverture.
- Le bouton principal est clair et accessible au pouce.
- Le texte visible est court et utile.
- Les actions secondaires ne noient pas l’action principale.
- La navigation entre pages métier n’est pas cassée.
- La session ne renvoie pas inutilement vers le PIN.
- Aucun message technique ne fuit côté façade.
- La hiérarchie visuelle tient sur petit écran.

## Doctrine mobile DIGIY
- Un écran mobile n’est pas un desktop compressé.
- Moins de blocs, plus d’action.
- Le cœur métier doit apparaître très vite.
- Les réglages profonds doivent rester derrière.

## Sortie
Donner :
1. les problèmes visibles,
2. les corrections prioritaires,
3. si demandé, une recouture directe du fichier.
