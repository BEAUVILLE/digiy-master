# SKILL — Audit Mobile Terrain (DIGIY)

## Priorités core
- Moins de texte, plus d'action
- Action principale visible immédiatement sans scroll
- Wording humain, zéro jargon technique

## Ce qu'on vérifie
1. Clarté écran d'ouverture (moins de 3 secondes)
2. Visibilité CTA principal (above the fold)
3. Densité texte (trop = problème)
4. Friction session/accès (PIN loop, redirect, flash blanc)
5. Confort tactile (zones de tap min 44px)

## Ce qu'on ne fait PAS
- Ne pas réécrire le backend
- Ne pas inventer un nouveau rail si le rail réel existe
- Ne pas déclencher un redesign complet pour un fix ciblé

## Format de sortie
1. Diagnostic rapide
2. Bloquants identifiés
3. Fixes prioritaires
4. Ce qu'on laisse intact
